Rule Sources Three families, one audit contract

Core Rules

System-maintained baseline

Regex + AST rules mặc định do hệ thống định nghĩa. Có thể tắt/bật hoặc chỉnh penalty nhưng vẫn là mặt sàn chất lượng dùng chung.

AI-Only Rules

Deep-audit-only reasoning

Những rule logic như `SILENT_DATA_CORRUPTION` hay `INSECURE_RANDOM` chỉ AI mới suy luận được, không ép regex hay AST.

NLRE Custom Rules

Natural language authoring

Prompt tiếng Việt được compile thành JSON rule. Thành công chỉ được công nhận khi sandbox và save flow đi hết contract.

Override Model Global baseline, project delta

Effective State

Override math must beat UI intuition

mergedDisabled = (global.disabled U project.disabled) \ project.enabled

Scope project có quyền bật lại rule bị Global disable. Điều này không thể suy ra đáng tin chỉ từ nút toggle trên màn hình; nó phải được định nghĩa bằng set operation rõ ràng.

Stored Fields
Field Meaning
disabled_core_rules Danh sách rule bị tắt ở scope hiện tại.
enabled_core_rules Chỉ meaningful ở project scope để override global disable.
custom_weights Penalty override theo từng rule.
Review Implication

KPI active count và Override Manager chỉ đúng khi stale rule IDs bị lọc trước. Nếu không, UI sẽ báo drift giả và làm sai quyết định của reviewer.

The real source of truth is merged state, not whichever tab is currently open. That is why override logic belongs in data shape, not just UI.
Builder Lifecycle Compile, test, save and reset

Primary APIs
GET /api/rules POST /api/rules/compile POST /api/rules/save POST /api/rules/reset
  • `compile` phải tách khỏi `save` để người dùng kiểm chứng signal trước khi chạm state thật.
  • `save` cần merge compiled JSON mới với phần custom đang tồn tại, không overwrite mù.
  • `reset` theo `target + level` tốt hơn việc xóa nguyên row vì cho phép rollback chính xác hơn.

UX Principle

Authoring phải cho cảm giác nhanh và thông minh, nhưng mọi hành động write xuống DB đều phải có semantics rõ: đang lưu cái gì, ở scope nào, và reset mức nào.

Good rule tooling feels creative on top and conservative underneath. That split is intentional, not accidental.
Guardrails The fixes that make the system trustworthy
05 / 06
Recent Correctness Fixes
  • Active KPI chỉ đếm rule thật sự còn tồn tại trong `rules.json`, tránh stale IDs làm lệch tổng.
  • `fetchRules()` đọc đúng `project_rules.custom_weights`, ngăn save vô tình xóa penalty overrides cũ.
  • `handleSave()` merge custom rule cũ và mới thay vì overwrite `compiled_json` toàn bộ.
  • Tab Custom render đủ `regex_rules`, `ast_rules`, `ai_rules`, không còn trạng thái lưu thành công nhưng UI báo rỗng.
Reset Strategy
  • Global và Project có level reset khác nhau để tránh “nuclear option” khi không cần.
  • Double-confirm + timeout làm giảm khả năng thao tác nhầm ở những khu vực nguy hiểm.
  • Override Manager hiển thị diff thực sự, không chỉ liệt kê raw DB fields.
Why This Matters

Với control plane, sai sót nhỏ ở merge hoặc reset logic không chỉ là bug UI; nó phá vỡ niềm tin vào audit result và khiến team ngại tùy biến rule trong thực tế.

Flexibility only becomes an asset after safety nets exist. Without them, the system becomes impossible to trust.
